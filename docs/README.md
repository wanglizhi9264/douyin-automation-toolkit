# 文档索引

本目录保存可以长期复用的产品、设计、接口和发布资产。聊天记录、临时日志和个人下载数据不属于事实源。

## 当前事实源

| 文档 | 作用 | 何时必须更新 |
| --- | --- | --- |
| [产品规格](product-spec.md) | 用户目标、功能边界、验收条件 | 新增入口、改变行为或验收口径 |
| [下载架构设计](architecture-design.md) | 组件职责、数据流、状态和恢复语义 | 修改接口、状态键、文件结构或下载逻辑 |
| [测试与发布手册](test-release-runbook.md) | 自动检查、人工冒烟、发布和故障取证 | 新增测试、发布物或诊断文件 |
| [抖音访问面说明](douyin-access-surfaces.md) | 已验证接口、执行上下文、失败模式 | 接口、字段、请求方式或风控表现变化 |
| [根规格入口](../SPEC.md) | 项目历史规格与当前规格入口 | 规格体系或里程碑变化 |

当前浏览器插件版本：0.4.0。

## 文档优先级

1. 代码和自动测试说明当前实际行为。
2. 产品规格定义对用户的承诺。
3. 架构设计定义实现不变量和修改边界。
4. 接口说明、测试手册提供操作细节。
5. 根目录 `SPEC.md` 的历史章节不覆盖当前事实源。

代码与规格不一致时，先判断代码缺陷还是规格变更，再同时更新实现、测试和文档。

## 变更沉淀规则

- 用户行为变化：更新产品规格。
- 接口、分页、重试或消息类型变化：更新访问面说明和架构设计。
- IndexedDB、文件夹记录或路径变化：更新架构设计和迁移说明。
- 新增失败模式或日志：更新测试与发布手册。
- 插件版本变化：同步 `manifest.json`、`package.json` 和文档版本。
- 新增自动测试：纳入 `npm run check` 并更新测试矩阵。
- `dist`、下载视频、Cookie、浏览器 Profile 和个人日志不得提交 Git。

## 维护原则

- 规格必须可测试，避免只写愿景。
- 设计记录关键取舍和不变量，不复制整段实现代码。
- 示例必须脱敏，不包含 Cookie、Token、真实备份目录或个人作品。
- 路径使用仓库相对路径，不写个人电脑绝对路径。
- 对外部扩展只沉淀经授权学习到的行为和架构，不提交第三方发布包或源码副本。
