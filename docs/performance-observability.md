# 性能与可观测性方案

## 1. 目标

本资产回答两个不同问题：

1. 为什么参考扩展打开后能很快显示喜欢和收藏总数。
2. 当前任务慢在抖音接口、主动限流、详情请求、媒体网络、磁盘写入还是插件自身渲染。

插件 0.4.0 不采集第三方扩展内部数据，也不复制其实现。若要比较第三方扩展，只能用相同账号、相近时段、同一网络和浏览器开发者工具做外部黑盒观察。

## 2. 快速总数

侧边栏打开后调用一次 `GET_SELF_PROFILE`。响应中的 `favoriting_count` 和收藏统计字段用于显示“抖音喜欢/抖音收藏”，不需要先分页扫描全部列表。

三类数量必须分开：

| 数量 | 来源 | 语义 |
| --- | --- | --- |
| 抖音官方总数 | 个人资料接口 | 当前账号公开给自己的总量快照 |
| 已检查数量 | 当前分页状态机 | 本轮实际扫描并归一化的视频数 |
| 已下载数量 | IndexedDB 与文件夹记录 | 已成功写入目标目录的作品数 |

官方总数可能包含图文、失效作品或接口暂时不可下载项目，因此不能当作本轮必然可下载数量。

## 3. 阶段定义

| 阶段 | 说明 | 高时优先判断 |
| --- | --- | --- |
| `profile_api` | 个人资料请求 | 页面桥、登录态、接口网络 |
| `list_api` | 喜欢/收藏单页接口 | 抖音接口、验证码、网络 |
| `list_throttle_wait` | 页间主动等待 | 风控保护，不是网速 |
| `list_retry_wait` | 失败后的退避 | 上游失败导致的保护等待 |
| `detail_api` | 作品详情请求 | 候选缓存未命中或封面缺失 |
| `download_candidates_reused` | 列表候选直接复用 | 命中越多，重复详情越少 |
| `video_request` | 发起媒体请求到收到响应头 | CDN 首包与连接 |
| `video_transfer` | 响应头到完整 Blob | 实际下载带宽 |
| `video_write` | Blob 写入授权目录 | 磁盘、杀毒软件、目录类型 |
| `checkpoint_write` | 写 `download-state.json` | 每条都执行，必须轻量 |
| `artifact_full_write` | HTML、数据库和完整日志 | 每 10 条或 30 秒，暂停/结束强制 |
| `item_delay` | 条目间随机保护等待 | 主动等待，不是网络问题 |
| `ui_render` | 侧边栏刷新 | 过高说明 UI 更新过密 |

## 4. 输出契约

每次下载任务在授权目录生成：

`data/.appdata/performance-summary.json`

关键字段：

- `wallMs`：任务墙钟总耗时。
- `sampleCount`：阶段样本数。
- `stages.<name>`：count、totalMs、averageMs、p50Ms、p95Ms、maxMs、failures。
- `stages.video_transfer.bytes` 与 `megabytesPerSecond`：媒体传输总量和平均吞吐。
- `bottlenecks`：按阶段累计耗时排序的前五项。

侧边栏只显示摘要；完整判断以该 JSON 为准。

## 5. 判读规则

- `video_transfer` 高且 MB/s 低：主要是网络或 CDN。
- `video_request` 高、传输快：主要是连接或 CDN 首包。
- `detail_api` 次数接近下载条数：检查候选是否过期、列表是否缺画质字段或是否因封面触发详情。
- `list_throttle_wait` 是最大项：流程受主动限流保护约束，不能简单提高并发。
- `checkpoint_write` 高：检查目标是否为机械盘、网络盘、同步盘或被实时扫描。
- `artifact_full_write` 每条都出现：属于实现回归；正确节奏是每 10 条、30 秒或任务安全点。
- `ui_render` 高：检查是否恢复成每条日志立即全量 render。

## 6. 可重复对比方法

1. 固定浏览器版本、账号、网络和目标磁盘。
2. 分别清空测试目录，但不要清除真实备份。
3. 记录打开侧栏到官方总数出现的时间。
4. 选择同一批 10 个可下载视频，记录总墙钟时间和阶段汇总。
5. 至少重复 3 次，比较 P50，不用单次最好值。
6. 第三方扩展只记录外部可见的网络请求数量与耗时，不导出 Cookie、Token、源码或个人数据。

验收目标不是保证绝对速度，而是能明确回答时间花在哪一步，并保证优化不破坏最高画质、账号隔离、限流和断点续传。
