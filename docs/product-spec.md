本文是当前可执行规格。根目录 `SPEC.md` 中的早期里程碑仅用于历史追溯。

## 1. 产品目标

产品让用户在已登录的 Chromium 浏览器中备份自己的抖音数据：

- 获取“喜欢”和“收藏”视频列表。
- 下载可获得的最高画质视频，并在候选不可用时安全回退。
- 将视频、封面、元数据、诊断日志和续跑记录写入用户授权目录。
- 中断、刷新或重新打开浏览器后，继续同类任务并跳过已成功下载的作品。
- 错误可见、可定位、可恢复，不把空列表误报成完成。

## 2. 范围与非目标

当前范围：

- Chromium Manifest V3 浏览器扩展。
- 当前登录用户的喜欢、收藏和已有关注列表下载入口。
- 用户主动授权的本地文件夹。
- 视频、可选封面、单条 manifest、汇总数据库、状态和日志。
- 收藏动作与审计能力继续保留，下载备份是当前优先链路。

非目标：

- 不绕过验证码、风控、限流、登录或平台权限。
- 不下载用户无权访问的私密、失效内容。
- 不保证图文、直播回放、合集等非普通视频内容可下载。
- 不向远程服务器上传视频、Cookie、Token 或个人进度。
- 不复制第三方闭源扩展代码；只复用经授权学习的行为模式并独立实现。
- 不承诺抖音未公开 Web API 永久稳定。

## 3. 用户故事

### US-01 选择并记住目录

用户选择可读写文件夹后，插件记住 File System Access 句柄。后续任务优先复用该目录；权限失效时明确要求重新授权。

### US-02 下载喜欢视频

点击“下载喜欢视频”后，从第一页边检查边下载，自动跳过已成功作品，直到可信终止游标。

### US-03 下载收藏视频

点击“下载收藏视频”后，按收藏接口分页读取，跳过图文、重复项和已下载收藏视频，复用喜欢视频的媒体下载与最高画质逻辑。

### US-04 继续下载

点击“继续下载”后，优先读取文件夹 `download-state.json` 的 `current.scope`，不存在时读取 `lastDownloadScope`。列表从第一页重新核对，以 `downloaded` 记录幂等跳过，不依赖可能过期的远端游标。

### US-05 获取可诊断结果

用户能看到北京时间日志，并在授权文件夹中取得下载状态、汇总库、静态报告和诊断日志。

## 4. 功能要求

### FR-01 登录态与账号隔离

- 抖音 JSON API 请求必须从 `www.douyin.com` 页面上下文发起。
- 开始喜欢或收藏下载前必须获取 `uid` 或 `secUid`。
- 文件夹记录若属于另一个账号，必须停止写入并提示选择新目录。
- 未登录、非 JSON、预计有内容但全程返回空列表时必须报错，不能显示“没有待下载”。

### FR-02 下载目录

- 主下载任务必须优先使用用户授权文件夹，不静默降级到浏览器下载记录。
- 文件夹句柄应被记住并在后续任务中恢复权限。
- 用户取消选择时必须显示明确原因。
- 文件夹 `download-state.json` 是该目录续跑的首要事实源。

### FR-03 喜欢列表

- 使用 `GET /aweme/v1/web/aweme/favorite/`。
- 每次任务从第一页扫描到状态机确认终止。
- 项目标记为 `liked` 或 `favorite_api` 作用域。
- 已下载项必须跳过；`failed` 和 `not_started` 可在后续任务重试。
- 预计喜欢数大于 0 而接口全程返回 0 条时必须报错。

### FR-04 收藏列表

- 使用 `POST /aweme/v1/web/aweme/listcollection/`。
- `Content-Type` 为 `application/x-www-form-urlencoded`。
- 请求体包含 `count=10` 和 `cursor=<当前游标>`。
- 首次 `cursor=0`；相邻页面请求至少间隔 3000ms。
- 单页最多尝试 3 次；前两次失败分别等待 7 秒、14 秒。
- 单次页面消息超时 60 秒。
- 使用 `aweme_list`、`cursor`、`has_more`；`total/total_count` 只作预计总数。
- 图文和本轮重复 `awemeId` 必须跳过。
- `has_more=false` 但 `cursor` 非 0 时继续请求。
- 只有 `has_more=false` 且 `cursor=0` 才是完整结束。
- 游标不前进、以 `-1` 或缺失游标异常结束时停止并记录，不能误报完成。

### FR-05 作用域与幂等

- 同一 `awemeId` 可以同时出现在喜欢和收藏。
- 记录主键必须包含作用域，格式为 `<scope>:<awemeId>`。
- `favorite_api` 归一到 `liked`。
- `liked` 与 `bookmarked` 的状态、路径和错误必须独立。
- 文件夹恢复必须用 `source + awemeId` 匹配，不能只按 `awemeId`。

### FR-06 最高画质

- 列表归一化时必须保存最高画质排序候选、普通兼容候选和候选获取时间。
- 开启最高画质时，10 分钟内的列表候选可直接复用；候选缺失或过期时才获取最新作品详情。
- 开启封面下载但列表没有封面时允许获取作品详情补齐封面。
- 候选复用不得改变最高画质排序和最多 4 个去重候选的回退规则。
- 候选来自 `video.bit_rate`、`play_addr_h264`、`play_addr_h265`、`play_addr`。
- 最高画质首先比较像素面积，再比较码率、大小、帧率和质量类型，同等条件可轻微偏好 H.265。
- 普通模式优先 H.264。
- 文件夹模式最高画质最多尝试排序后的 4 个去重候选。
- 下载结果必须校验 2xx、非 206、`video/mp4` 和非零内容。
- 高质量候选不可用时可回退兼容候选，但必须写日志。
- 最终 codec、分辨率、码率、帧率、大小和候选序号进入记录或 manifest。

### FR-07 文件写入

- 喜欢视频：`data/点赞/视频/<awemeId>.mp4`。
- 收藏视频：`data/收藏/视频/<awemeId>.mp4`。
- 封面写入各作用域的 `封面` 目录。
- 单条 manifest：`data/.appdata/manifests/<awemeId>.json`。
- 状态：`data/.appdata/download-state.json`。
- 收藏汇总：`data/.appdata/db_bookmarked.json`。
- 日志：`data/.appdata/download-log.json`。
- 性能汇总：`data/.appdata/performance-summary.json`。
- 静态报告：`本地库.html`。
- 媒体写入成功后才能标记 `downloaded`。

### FR-08 暂停、继续与保护

- 暂停请求在当前安全点停止并保存状态。
- “继续下载”恢复上次 scope，重新扫描并跳过 `downloaded`。
- 连续 8 个视频下载失败时停止任务。
- 每条失败保存 `lastError` 和结构化日志。
- 网络离线时等待恢复，不快速耗尽重试。

### FR-09 可观测性

- UI 日志使用 `Asia/Shanghai` 时区。
- 日志包含适用的页码、游标、数量、作品 ID、候选序号、画质和错误。
- 落盘诊断过滤敏感键并截断超长字符串。
- 侧边栏打开后通过一次个人资料请求展示抖音官方喜欢数和收藏数；该值不冒充已扫描或已下载数量。
- 每个下载任务统计 profile、列表、详情、限流等待、媒体首包、媒体传输、磁盘写入、检查点、完整报表和 UI 渲染耗时。
- 性能汇总必须包含次数、总耗时、平均值、P50、P95、最大值、失败数；媒体传输同时记录字节数和 MB/s。
- 任务结束时把汇总写入授权文件夹，并在侧边栏显示主要瓶颈。
- “没有新的待下载项目”只能在完整扫描成功且所有项目已跳过后显示。

## 5. 本地数据结构

```text
<用户目录>/
├── data/
│   ├── 点赞/
│   │   ├── 视频/<awemeId>.mp4
│   │   └── 封面/<awemeId>.jpg
│   ├── 收藏/
│   │   ├── 视频/<awemeId>.mp4
│   │   └── 封面/<awemeId>.jpg
│   └── .appdata/
│       ├── download-state.json
│       ├── download-log.json
│       ├── performance-summary.json
│       ├── db_likes.json
│       ├── db_bookmarked.json
│       ├── db_authors.json
│       ├── db_videos.json
│       ├── db_texts.json
│       └── manifests/<awemeId>.json
├── 本地库.html
└── 说明.txt
```

IndexedDB 名为 `douyin_toolkit`，当前版本 2。业务作品写入 `scopedItems`；旧 `items` 仅用于兼容迁移。

## 6. 非功能要求

- 安全：不提交或导出 Cookie、Token、浏览器 Profile 和真实媒体。
- 可恢复：刷新、扩展重载或暂停后不重复下载成功项。
- 可诊断：所有终止都有用户可见原因和持久化日志。
- 节流：收藏分页不得快于 3 秒。
- 兼容：目标为支持 MV3 和 File System Access API 的 Chromium。
- 可维护：接口、状态、测试和文档在同一变更中更新。
- 本地优先：媒体和个人记录只保存在本机授权目录。

## 7. 验收条件

发布必须同时满足：

1. `npm run check` 全部通过。
2. ZIP 包包含 manifest、消息桥、`bookmarked-sync` 和下载模块。
3. 相同 `awemeId` 的喜欢/收藏状态互不覆盖。
4. 收藏 POST、表单、分页终止和重试常量有自动测试。
5. 文件夹直写、权限复用、下载记录恢复有自动测试。
6. 人工冒烟覆盖选择目录、喜欢、收藏、暂停继续、刷新后跳过已下载项。
7. `manifest.json` 与 `package.json` 版本一致。
9. `test-performance.mjs` 验证阶段聚合、候选复用、检查点批量刷新和性能文件契约。
8. Git 不包含 `dist`、下载媒体、个人日志或浏览器 Profile。
