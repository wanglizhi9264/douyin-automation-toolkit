# ADR-0002：继续下载采用从第一页重新对账

- 状态：Accepted
- 日期：2026-07-15
- 影响版本：插件 0.3.0+

## 背景

抖音列表会新增、删除和重排，远端分页 cursor 可能过期或改变含义。若把 cursor 当作长期断点，从中间恢复可能漏掉新作品，也可能把异常终止误认为完成。

## 决策

“继续下载”不恢复上次远端 cursor，而是：

1. 从文件夹 `download-state.json.current.scope` 恢复任务类型；没有文件夹记录时使用 `lastDownloadScope`。
2. 对目标列表从第一页重新扫描。
3. 使用本地 `downloaded` 状态幂等跳过已成功作品。
4. 只有列表状态机达到可信终止条件才标记完整完成。

收藏列表的可信终止条件是 `has_more=false` 且 `cursor=0`。

## 后果

正面：

- 新增或重排作品不会因旧 cursor 被漏掉。
- 远端 cursor 变化不会破坏本地下载断点。
- 文件夹记录可以跨页面刷新和扩展重载恢复。

代价：

- 继续任务会重复读取已经扫描过的页面。
- 必须严格节流并可靠跳过已下载项。
- 扫描总进度可能在重新对账时从第一页重新显示。

## 约束

- 不得只凭 `has_more=false` 提前结束收藏扫描。
- 文件夹记录优先于浏览器内上次 scope。
- 续跑语义变化必须同时更新产品规格、状态机测试和人工冒烟步骤。
